### 6Ô∏è‚É£ Revert the Fix and Confirm Failure

To confirm that the STIG compliance result was directly tied to the ECC Curve Order configuration, I manually reverted the setting using the Windows Registry Editor.

---

### Open Registry Editor
- Press **Windows key + R** to open the Run dialog box  
- Type `regedit` and press **Enter**

---

### Navigate to the ECC Configuration Key
In Registry Editor, navigate to the following path:

```
HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Cryptography\Configuration\SSL\00010002
```

---

### Revert the ECC Curve Order Configuration
- In the right-hand pane of the `00010002` key, locate the value named **EccCurves**
- **EccCurves** is a `REG_MULTI_SZ` value that defines the enforced ECC curve order
- Right-click **EccCurves** and select **Delete**
- Confirm the deletion

Deleting the **EccCurves** value removes the explicitly configured ECC curve order and returns the policy to a **Not Configured** (non-compliant) state.

### 7Ô∏è‚É£ Generate and Apply the PowerShell Remediation

After validating the manual fix, I translated the STIG requirements into an automated PowerShell remediation script.

This control is enforced through registry-based Group Policy values and local system cryptographic settings, so the script was designed to configure the required ECC Curve Order programmatically rather than relying on the Local Group Policy Editor.

The script was designed to:

- Enforce a STIG-compliant ECC Curve Order configuration  
- Configure the approved elliptic curves in the required order (NistP384, followed by NistP256)  
- Apply the configuration at the registry-based Group Policy level to ensure policy enforcement  
- Apply the configuration at the local Schannel ECC level to ensure consistent system behavior  
- Create required registry paths if they do not already exist  
- Verify the effective configuration after applying the changes  
- Recommend a system reboot to ensure cryptographic providers fully apply the updated settings
